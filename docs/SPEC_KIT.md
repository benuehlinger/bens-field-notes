# Spec Kit workflow for website development

Use Spec Kit only for **website/product** work in this repo.

## Why here

- UI and content structure have higher ambiguity than ingest code.
- Spec-first helps keep tone, IA, and chart requirements consistent.

## Suggested loop

1. `specify init . --ai <agent>`
2. `/speckit.constitution` - principles (source lines, as-of date, no hype copy)
3. `/speckit.specify` - define website behavior and page outcomes
4. `/speckit.plan` - lock architecture, routes, and data contract choices
5. `/speckit.tasks` - execution breakdown
6. `/speckit.implement` - implement in phases

## Constitution starter points

- Every chart includes source and as-of metadata.
- Sarcasm is allowed in narrative text, never in place of methodology.
- No hidden transforms: list units, smoothing, and window assumptions.
